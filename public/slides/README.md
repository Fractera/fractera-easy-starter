# Slides — изображения для секции LoopShowcase

Эта папка хранит изображения для секции `LoopShowcase` на главной странице (H2: «Полное владение стеком…»).

## Правило именования

Каждый файл — это номер слайда: `1.jpg`, `2.jpg`, `3.jpg`, …, `99.jpg`, `100.jpg`.

Номер совпадает с номером, который рисуется в кружке на UI и с порядком слайда в локали (`lib/i18n/locales/{ru,en}/loop-showcase.ts`).

## Как добавить новый слайд

1. Положите картинку в эту папку с именем `N.jpg`, где `N` — номер слайда.
2. Добавьте текстовый блок в `lib/i18n/locales/ru/loop-showcase.ts` и `lib/i18n/locales/en/loop-showcase.ts` в массив `slides` (по индексу `N-1`).
3. В каждом блоке должен быть `imageSrc: '/slides/N.jpg'`.
4. Готово — Vercel переразвернёт фронт автоматически после push.

## Как просить помощника

Если вы добавляете несколько слайдов сразу — скажите помощнику, например:

> «Я загрузил изображения 10, 11, 12. Вот тексты для них: …»

Помощник создаст для них записи в обоих локалях с `imageSrc: '/slides/10.jpg'`, `/slides/11.jpg`, `/slides/12.jpg` и привяжет ваши тексты.

## Технические требования

- **Формат**: `.jpg` (рекомендуется для фото / скриншотов). Можно `.png` — тогда в локали укажите `imageSrc: '/slides/N.png'`.
- **Размер**: 16:9 (например, 1920×1080). Меньше — допустимо, но изображение растянется до aspect-[16/9].
- **Вес**: до 300 КБ на картинку. Перед заливкой пропустите через [tinyjpg.com](https://tinyjpg.com) или `cwebp`.
- **Lazy-loading**: компонент сам подгружает только текущий + следующий блок (по 3 слайда), так что 100+ слайдов не нагружают старт страницы.
- **Blur placeholder**: до загрузки картинки показывается gradient + иконка. Когда `<img loading="lazy">` срабатывает `onLoad`, картинка плавно проявляется.

## Что произойдёт, если файла нет

Если `imageSrc: '/slides/N.jpg'` указан, но файла нет — браузер просто не загрузит картинку, и слайд останется на placeholder (gradient + иконка). Ничего не сломается.
