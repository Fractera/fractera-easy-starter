# Шаг 1 — Аудит деплоев + дашборд пользователя

## Что было сделано

**Проблема 1 — Admin теряет контроль после «Перезапустить»**
Строка исчезала из таблицы «Ошибки деплоя» сразу после клика. Не было истории: кто запустил, когда, что получилось.

**Проблема 2 — Пользователь в темноте**
После ошибки деплоя пользователь не видел ни статуса, ни того, что проблема устраняется, ни информации о подписке.

### Реализовано

1. **`prisma/schema.prisma`** — новая модель `DeployAttempt`: записывает каждую попытку деплоя (triggeredBy, status, error, subdomain, serverIp, startedAt, completedAt). Добавлена связь в `ServerToken`.

2. **`app/api/stripe/webhook/route.ts`** — создаёт `DeployAttempt { triggeredBy:'webhook' }` перед запуском деплоя; при ошибке в catch обновляет статус на `failed`.

3. **`app/api/admin/redeploy/route.ts`** — создаёт `DeployAttempt { triggeredBy:'admin' }` перед перезапуском; при ошибке обновляет на `failed`.

4. **`app/api/progress/route.ts`** — завершает `DeployAttempt` по `deploySessionId`: `success` при получении субдомена, `failed` при ошибке или отсутствии субдомена.

5. **`app/api/admin/deploy-history/route.ts`** (новый) — `GET ?serverTokenId=xxx` возвращает все попытки для токена.

6. **`app/api/admin/servers/route.ts`** — добавлена ветка `status=deploy-issues`: возвращает `status=error` И `status=pending` с admin-редеплоем в процессе.

7. **`app/api/my-servers/route.ts`** — добавлен `isRedeploy: deployAttempts.count > 1`.

8. **`app/admin/page.tsx`** — секция переименована «Ошибки деплоя» → «Деплои». Новый компонент `DeployRow` с двумя режимами: `status=error` → форма IP/пароль + перезапуск; `status=pending (isInProgress)` → история попыток + форма. Строка НЕ исчезает после нажатия «Перезапустить» — показывает «🔄 Устраняется» и историю.

9. **`components/dashboard-modal.tsx`** — добавлены вкладки «Серверы» / «Подписка». Исправлен `isStale` (не срабатывает на редеплой). Улучшены статусы: `error` → оранжевый с текстом «Не удалось установить автоматически — наша команда устраняет проблему»; `pending+isRedeploy` → синий «Устраняем» + «установка запущена повторно». Вкладка «Подписка» показывает тариф, дату окончания периода, статус.
